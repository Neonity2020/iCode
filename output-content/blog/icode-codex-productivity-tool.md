# 我们用 Codex 做了一个开源 Desktop：iCode

过去一段时间，我越来越强烈地感觉到一件事：

**Codex 不只是一个 AI 编程工具。**

很多人第一次接触 Codex，是让它写代码、改 bug、跑测试、解释项目。这个入口当然成立，但如果只把它理解成“更强一点的 AI 程序员”，其实低估了它。

更准确地说，Codex 是一个可以操作工具、理解上下文、持续执行任务的生产力系统。

它能写项目，也能做深度研究；能改代码，也能整理资料；能读文档、跑命令、查证信息、生成报告、推进一个复杂任务从想法到结果。

换句话说，它不只是帮你“写代码”，而是在帮你“完成工作”。

## Codex 的真正价值，是工作流

传统 AI 聊天工具的问题在于，它很容易停留在“问答”里。

你问一句，它答一句。  
你再补充一点，它再接着答。  
真正复杂的任务，往往还是要靠人自己拆解、复制、验证、整理。

但 Codex 不一样。

它天然更接近一个工作台：它可以进入你的项目，理解文件结构，调用命令，修改内容，保留上下文，持续推进任务。你给它的不是一句问题，而是一整个工作环境。

这就是为什么我认为 Codex 的边界远不止编程。

当你做深度研究时，它可以帮你搜集资料、整理观点、提炼结构、生成初稿。

当你做产品设计时，它可以帮你拆需求、写原型、做页面、跑起来验证。

当你做内容创作时，它可以帮你从材料里提炼主题，组织成文章、脚本或方案。

当你做日常工作时，它可以帮你处理那些需要读、查、写、改、验证的复杂流程。

它像一个能坐在你电脑前、理解任务、调用工具的协作者。

## 但我们还缺一个真正顺手的桌面工作台

Codex CLI 是开源的，而且非常强。

它适合在终端里启动，适合开发者快速进入任务，也适合自动化和脚本化流程。

但当我真的把 Codex 当成日常生产力工具来用时，我发现自己想要的不只是一个命令行界面。我想要一个更完整的桌面工作台：

能选择本地工作区。  
能看到文件树。  
能管理终端。  
能查看任务状态。  
能审阅变更。  
能保留多个工作区和会话。  
能把 Codex 的能力放进一个更接近真实工作的界面里。

官方公开的开源组件里，Codex CLI、Codex SDK、Codex App Server 等都可以在 GitHub 上找到；但完整的 Codex Desktop/App 客户端，并没有以同样的方式开源。

所以我们想：既然 Codex CLI 已经开源，为什么不基于它，vibe coding 一个开源 Desktop？

于是有了 iCode。

## iCode 是什么？

**iCode 是一个面向 Codex 的开源桌面工作台。**

它不是重新造一个 AI 模型，也不是重新实现一个智能体系统。它的核心思路很简单：

**把 Codex CLI 的能力接入到一个更适合真实工作流的桌面应用里。**

当前 iCode 已经支持：

- 本地工作区选择
- 文件树浏览
- PTY 终端
- Codex CLI app-server 接入
- Finder 与外部链接集成
- 官网 Landing Page 展示项目方向和快速开始

从工程结构上，iCode 使用 pnpm monorepo：

- `apps/desktop` 负责 Electron 桌面端
- `marketing` 负责官网 Landing Page
- `packages/app` 负责共享 React 产品层
- `packages/platform` 负责跨端平台能力契约

我们希望它既是一个可用的工具，也是一个可以被社区继续改造的起点。

## 为什么要开源？

因为这类工具不应该只属于一个封闭产品。

每个人使用 AI 的方式都不一样。有人希望它更像 IDE，有人希望它更像研究助理，有人希望它接入自己的知识库、工作流、脚本、内部系统。

如果桌面工作台是开源的，大家就可以真正按照自己的需求去扩展它。

你可以改界面。  
可以加工具。  
可以接自己的 MCP。  
可以做团队内部版本。  
可以把它变成研究工作台、内容工作台、项目管理工作台，甚至是你自己的 AI 操作系统雏形。

这也是 iCode 最想探索的方向：

**让 Codex 从一个编程助手，变成一个开放的生产力入口。**

## 用 Codex 做 Codex 的 Desktop

iCode 本身也是一次很有意思的实验。

我们用 Codex 来构建一个 Codex Desktop。  
用 AI 编程工具去创造更适合 AI 工作流的工具。  
用 vibe coding 的方式，把一个想法快速推进成真实可运行的应用。

这件事本身就说明了一个变化：

过去，做一个桌面软件可能需要完整团队、很长周期、复杂分工。

现在，一个人、一台电脑、一个清晰想法，再加上 Codex，就可以把产品原型一路推到可用状态。

这不是“AI 替代开发者”。

更像是开发者的半径被放大了。

## 接下来

iCode 还在早期，但方向已经很明确：

我们想把它做成一个本地优先、开源、可扩展的 Codex 工作台。

它首先服务编程，但不会止步于编程。

它首先是 Desktop，但未来也可以延伸到更多工作场景。

它首先是一个工具，但更长期看，它可能是一种新的工作方式。

Codex 不只是 AI 编程工具。  
Codex 是生产力工具。  
而 iCode，是我们为这个判断做出的一个开源实践。
